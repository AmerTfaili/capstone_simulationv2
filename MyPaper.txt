Algorithm 1 Phase-wise Budgeted Critical Node Selection
and Control
Parameters: Traffic graph G = (V, E), budget B, number
of phases n
Returns: Final controlled subset Sn, trained shared policy
πθ
1: Initialize controlled subset S1 such that |S1| = B
2: Initialize shared RL policy πθ and GATv2 forecaster gψ
3: for k = 1 to n − 1 do
4: Fix controlled subset Sk
5: if k = 1 then
6: Train πθ in SUMO on Sk using the base reward
and collect traffic logs
7: else
8: Freeze gψ for Phase k
9: Train πθ in SUMO on Sk using attention-based
reward shaping from frozen GATv2 coefficients
10: Collect traffic logs from the current phase
11: end if
12: Freeze πθ
13: Train gψ offline on the collected logs for queue-length
prediction
14: Run inference to extract predicted queue lengths, at-
tention weights, and node embeddings from gψ
15: Compute node criticality scores
16: Update Sk+1 under budget B using ranked scores and
bounded exploration
17: end for
18: return Sn and πθ
